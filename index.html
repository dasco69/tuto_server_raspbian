      -la version Lite (c’est pour les barbus qui kiffe le terminal)
      - la version Graphique (oui ya les menus des icônes trop cool)

Les différences :


V. lite, :
On accède au Raspbian à distance depuis ton PC, donc pas de souris.
Tu accèdes depuis ton terminal Linux ou Mac, pas Windows.
Os moins lourd.

V. graphique:
Tu peux te brancher en HDMI sur un écran.
Donc il y a une souris et un beau fond écran…
Si tu as Windows tu devras utiliser ton Raspbian branché à un écran avec clavier et souris brancher dessus.
Os plus lourd

Créer une carte Bootable.

Tu récupères donc ton .image qui correspond à ton image Raspbian Strech (Lite). Dans un OS tu as énormément de dossier contenant des fichiers. Nous allons juste utiliser certains fichiers qui vont forcer le PC (ici le RPI) à lancer les programmes se situant dans ces dossiers.

Explication :

En face To face tu ne vois qu’un fichier .IMG, c’est normal, les dossiers/fichiers son empiler pour ne générer qu’une seule image. Cela s’appelle de l’empilement. Je te la fais courte c’est pour comprendre l’idée, que tu ne fasses pas ça bêtement.

Revenons à nos claviers.

On va utiliser un logiciel pour créer une carte bootable . Tu peux utiliser divers logiciels :

      W32 Disc imageait
      UNETbootin
      ...

Ils marchent tous plus ou moins de la même manière. Certains n’acceptent pas certains formats. Donc juste ce référer à au doc.
Prenons ici UNETbootin, c’est super simple :

Disk image - format .image - retrouve le chemin ou se situe ton OS Type - retrouve ta carte micro SD

Fini en validant sur OK.

FINI
Tu plug ta carte dans le RPI et une fois branché comme indiqué plus haut, tout est fonctionnel.
(Sauf version Graphique, c'est presque fonctionnel.)

Mais où est Raspbery Pi?

Il faut à présent retrouver l’adresse de ton raspberry pi (si tu tournes avec un mode graphique sautes cette étape) Tu vas donc chercher lip local de ton RPI. $ ifconfig
Ce qui va nous interesser ici sera la partie eth0, Inet addr. L'ip local sera par exemple: 192.168.0.1. Voici l'ip de votre modem.
À partir de maintenant débrancher la prise de courant de votre RPI.(Vous comprendrez plus tard, pas le temps là ^^) On va utiliser à présent NMAP, qui est un outil très puissant pour scanner un réseau(les ports, ip...). sudo apt-get install nmap Ou celle-ci pour une distribution à base de RPM : sudo yum install nmap Derrière un petit: sudo apt-get update Si il y a besoin: sudo apt-get upgrade OK, maintenant utilisons NMAP. nmap -sn 192.168.0.1/24 nmap pour appeler le logiciel, -sn va pinger et scanner le réseau, 192.168.0.1 le réseau à scanner (ici le modem).
Regarder bien le nombre et la fin de chaque IP.
Rebranchez à présent votre RPI. Puis re-faite le même processus. nmap -sn 192.168.0.1/24 L'ip qui vient de ce rajouter correspond à votre RPI. Merveilleux.

On va enfin pouvoir se connecter à son RPI ?

Yes of course, une petite clope et on si met.

La notion SSH / FTP / SFTP NTM....

Franchement ce n'est pas compliquer faux juste si pencher un peu.
Tout est dit dans le nom.

  • SSH: Secure Shell
  • FTP: Files Transfert Protocol
  • SFTP: Secure Files Transfert Protocol

SHH te permet de naviguer sur ton serveur.
FTP te permet d'importer ou exporter sur ton serveur.
SFTP pareil que FTP mais de manière sécurisé.
À ma connaissance d'origine on peut seulement utiliser SSH et SFTP sur raspbery. La commande FTP a été supprimée pour des raison de sécurité.
C'est repartie. On se connecte à notre RPI. ssh pi@192.168.0.12 ssh pour le protocole pi@ le nom du serveur 192.168.0.12 l'ip du RPI (elle peut être différente chez vous).

Instalation du Serveur.

Il existe différents logiciels qui te permettront de faire un serveur, ici on parlera du plus connu Apache.
Tu as peut être entendu parler de WAMP , LAMP , MAMP ? C'est tout simplement un acronyme pour désigner ; Microsoft / Apache / MySql / Php, pareil pour Linux et Mac.

Un serveur va donc se composer de ses différents langages / logiciels qui te permettent de lire et paramétrer les fichiers, notamment gérer la base d’accès et le droit admin / client.
